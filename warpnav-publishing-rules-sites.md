- 指定标签

## 2. 正文内容规范

**字数要求**：1000-1500+ 字中文，内容详实丰富

### 段落结构（推荐）

```
第1段：网站简介（1-2句话概括定位）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
第2段：H2「主要功能」或「核心功能」
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
第3段：H2「主要特点」或「核心特点与优势」
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
第4段：H2「适用人群」或「适用场景」（可选）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
第5段：总结与推荐建议
```

### 排版要求（关键！）
- **H2/H3 标题**分隔主题（主要功能、主要特点、适用人群等）
- **粗体（strong）+ 列表**组合是核心排版模式
- 使用 **表格** 展示对比信息（如：与竞品的功能对比）
- 使用 **ul/li** 列表列举功能点
- 用 **strong** 强调关键词和关键数据
- 包含具体 **数据/数字**（如：10万+图标、每月更新等）
- 避免大段纯文字，每个功能点都用列表展示

### 排版示例
```html
<h2>主要功能</h2>
<ul>
<li><strong>功能一：</strong>具体说明...</li>
<li><strong>功能二：</strong>具体说明...</li>
<li><strong>功能三：</strong>具体说明...</li>
</ul>

<h2>核心特点</h2>
<table>
<tr><th>特点</th><th>说明</th></tr>
<tr><td><strong>特点一</strong></td><td>详细说明...</td></tr>
<tr><td><strong>特点二</strong></td><td>详细说明...</td></tr>
</table>
```

### 内容要点
- 开头1-2句话简短概括网站定位
- 每个功能点都要有具体说明，不说空话
- 常包含"适用人群"或"适用场景"总结
- 结尾有评价性总结和推荐建议
- 引用具体数据和实例增强说服力

## 3. 站点信息自定义字段

| 字段名 | 填写内容 | 说明 |
|--------|----------|------|
| `_sites_link` | 网站 URL | 链接地址 |
| `_sites_sescribe` | 15-30字中文短句| 一句话简介 |
| `_sites_language` | 如 `zh,en` | 根据网站语言 |
| `_sites_country` | 默认填 `USA` | 不确定时使用 |
| `_sites_preview` | 图片地址 | 网站预览图。使用 Puppeteer 截图服务自动获取 → 上传到 WP 媒体库 → 填写图片地址 |
| `_thumbnail` | favicon 图片地址 | 站点 LOGO 图标。使用 `https://t3.gstatic.com/faviconV2?client=SOCIAL&type=FAVICON&fallback_opts=TYPE,SIZE,URL&url=%url%&size=128` 获取（%url% 替换为网站地址，size 默认 128；如获取失败则改为 64）→ 上传到 WP 媒体库 → 填写图片地址 |
| `_seo_metakey` | 2-3 个关键词 | SEO 关键词 |
| `_seo_title` | 留空 | 不填写 |
| `_seo_desc` | 留空 | 不填写 |

### 网站预览图要求
- **尺寸**：约 1280×720 px（自动截图）
- **大小**：不超过 150KB
- **获取方式**：使用 Puppeteer 截图服务自动获取
  ```bash
  cd /home/satomini/.openclaw/workspace/services/screenshot
  node capture.js <网站URL>
  ```
- **处理流程**：截图 → 自动压缩 → 上传到 WP 媒体库 → 获取 URL 填入 `_sites_preview` 字段

## 4. 其他字段

- **分类目录**：用户指定
- **标签**：用户指定
- **未提及字段**：留空或保持默认

## 5. 发布状态规则

- **所有文章**：以**草稿（draft）**形式保存，等待审核后发布
- **发布审核**：由管理员审核内容后手动发布
- **状态说明**：创建时设置 `status: draft`，审核通过后改为 `publish`
